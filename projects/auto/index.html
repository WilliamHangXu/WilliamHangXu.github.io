Algorithms for Autonomous Vehicles

Introduction

This project implements the software and algorithms of an autonomous vehicle system that drives in a simulated mini city. The system commands the steering and acceleration of a realistic vehicle model using only raw sensor measurements and the output of various detection algorithms. See the GitHub repository for more detail.

System Structure

The system contains three modules: localization, perception, and motion planning. The goal is to drive the vehicle smoothly without running out of the road or colliding with other objects.

Localization The localization module gives the state of the vehicle. Rough state estimations are obtained from simulated sensors such as GPS and IMU, and then are fed to Extended Kalman Filter for more accurate state estimations. These information instruct both the perception module and the motion planning module.

Perception The purpose of perception is to accurately estimate the state of other objects on the road (other vehicles, pedestrians, etc.). The two simulated cameras on the vehicle gives 2D bounding boxes of the objects on the road. This information and the vehicle's state from localization are fed into Extended Kalman Filter for predicting the state of the objects, which includes their position and size.

Motion Planning For motion planning, a model predictive control algorithm is used. Trajectory are obtaind by solving a constrained optimization problem with the IPOPT solver. The objective function and constraints make use of information provided by localization and perception.
