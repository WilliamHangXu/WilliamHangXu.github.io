Protein Information Retriever

Introduction

Analyzing a protein's sequence is crucial for studying its structure and conformation change. This project aims to retrieve protein sequence data, feed it into various algorithms for richer information, and then gather and store the results in a data structure that' convenient for further analysis. See the GitHub repository for more detail.

Many thanks to Dr. Hassane Mchaourab and the lab members for their kind instruction and support!

Project Structure

Given a protein of interest, the program first downloads a PDB file for this protein from the Protein Data Bank, which includes its sequence information and coordinates of its atoms. This file is then fed to the DSSP (Dictionary of Secondary Structure of Proteins) algorithm to get the secondary structure of the protein. We also extract the primary sequence from the PDB file and feed it to the TOPCONS server to get the membrane affiliation information. The primary sequence is also fed to a sequence of format converting programs to get the sequence in a certain format, which is then fed into the Consurf server to obtain conservation information. All these information are then stored into a Protein object, which is a sequence of AminoAcid objects that store the amino acid's name, index, and all the information we obtained.
